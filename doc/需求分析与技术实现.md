# 需求分析 #

- 产品需要的内容和效果？
- 技术选型
- 技术指标

# 产品定型 #

- 包含登录/注册/找回密码/信息设置与修改
- 支持PC和移动端，各个子网站需要有自己的样式
- 功能逻辑必须统一和同步

# 技术方案 #

- 通过`JS SDK`的方式，由团队统一开发维护，保证功能的统一以及修改的同步。
- 支持PC和移动端，因此包体积要小，要分包，不能有依赖。
- `JS SDK`要包含全部业务逻辑，但不包含具体样式，由下游业务方进行自定义。

# 前端技术指标 #

- 浏览器兼容到IE8
- 支持PC和移动端，大小不能超过**30KB**
- 支持多种引用方式：直接引用，`commonJS`、`AMD`
